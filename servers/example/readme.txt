Emoji Log
Users can choose from one of five emoji, leaving behind a log of emoji choices.

This example shows:
 * scripting of required DB tables and columns
 * a static HTML app using the tiny servers API
 * a handful of API functions
 * different identity requirements, eg. public, logged in, and admin
 * user facing error messages

Home page
View the log (or most recent <X> item), from most recent to oldest, each entry is the date/time, email and emoji
Sign in button
Add an emoji function (you must be signed in)
Clear the list (you must be signed in as a recognised administrator)
